# Gene Report ask

## What to offer

The Gene Report Card is a free, personalized analysis of the recipient's specific genes. No call required, no commitment. The entry point is a 2-minute survey asking which genes their panel covers.

Lead with the report, not Qorus. The report demonstrates value directly. The discovery call is a natural next step after they receive and engage with it — not a cold ask in the first email.

## The four deliverables

Offer these as a menu on first touch (or pick one if you can pin the gene-stage — see `context/stage-offer-mapping.md`):

- **ClinVar discrepancy report** — conflicting submissions likely resolvable with evidence already in the literature
- **Reclassification candidates** — variants where existing evidence may support a classification update
- **VUS triage queue** — ranked by evidence completeness so you know where to focus first
- **Per-paper evidence summaries** — specific findings from relevant papers, mapped to ACMG criteria (hold back on first touch unless literature review is clearly the stated bottleneck)

Use the compressed phrasings above verbatim in bullet lists. Do not paraphrase — the specificity is what makes the offer credible. Full descriptions are in `docs/gene-report-fact-sheet.md` in `deepgene-operations` if needed for context.

## Default first-touch offer set

Surface the first three (ClinVar discrepancy, reclassification candidates, VUS triage queue) and let the recipient pick. Per-paper evidence summaries are the heaviest deliverable — hold unless the recipient's situation specifically calls for it.

## The ask

First touch: link to the 2-minute gene survey. Frame as "no call required, no commitment."

After report delivery: soft Calendly link — "a few findings stood out, worth 20 minutes to walk through?" That is the bridge to the discovery call.

Do not ask for a discovery call before the report has been delivered.

## Forward-looking hook (use in later emails or graceful exits)

The ClinGen V4 Bayesian framework is coming. When it arrives it will be the most significant retooling of variant classification in years. Qorus is being built to support that transition. Researchers who engage now have direct input into how that capability is shaped. Use this when someone declines now but might re-engage around the V4 launch.
